# VGA_CoinPusher

Unityで制作された、物理演算ベースのシンプルなコインプッシャーゲームのプロトタイプです。

## 🚀 概要
このプロジェクトは、物理的な相互作用、リソース管理、およびコンポーネント間の疎結合な通信に焦点を当てた、クラシックなコインプッシャーのコアメカニクスを実装しています。

## ✨ 主な機能
- **動的なコイン生成**: 指定されたエリア内への初期一括生成と、プレイヤー入力によるリアルタイム生成。
- **物理ベースの動作**: Rigidbody物理を使用したプッシャーの動きにより、リアルなコインの押し出しを実現。
- **リソース管理**: プレイヤーのコイン残高を管理し、「Finish」エリアにコインが落ちた際に自動更新。
- **インタラクティブな工夫**: プッシャーとの接触時にコインが滑らかに動くよう、動的な親子関係の構築ロジックを実装。

## 🛠 技術的な特徴

### 設計パターン: 簡易サービスロケーター
`CoinMessage`という静的ブリッジクラスを介して、`CoinCounter`と他のシステム（`CoinFactory`や`CoinPhysics`）間の通信を行っています。これにより、各プレハブがUIやカウンターオブジェクトへの直接的な参照（ハード参照）を持たずに、残高の更新や取得が可能になっています。

### 物理挙動の安定化
`CoinPhysics`スクリプトでは、TriggerとCollisionを組み合わせたハイブリッド方式を採用しています。コインがプッシャーの範囲内（Trigger）にあり、かつ物理的に接触（Collision）している間、一時的にコインをプッシャーの子オブジェクトにします。これにより、Unityの物理演算で発生しがちな「高密度な物体を押し出す際のガタつき」を抑え、滑らかな動作を実現しています。

## 📄 スクリプト解説

### `CoinCounter.cs`
プレイヤーのコイン残高を管理します。
- **設計意図**: ゲームの状態管理とUI更新の集中化。
- **仕組み**: 残高変更時にUI（Text）を更新し、自身を`CoinMessage`に登録します。

### `CoinFactory.cs`
コインの生成を担当します。
- **設計意図**: 初期状態（テーブル上のコイン配置）とプレイヤー操作の両方を管理。
- **仕組み**: 
    - `Start()`: ギズモで視覚化された範囲内にランダムにコインを生成。
    - `Update()`: スペースキー入力を検知し、残高を消費してコインをプレイヤー位置に生成。

### `Pusher.cs`
プッシャーの往復運動を制御します。
- **設計意図**: 一貫性のある物理法則に則った機械的な力の提供。
- **仕組み**: `Mathf.Sin`を用いて滑らかな往復経路を計算し、`Rigidbody.MovePosition`で移動させることで、物理衝突を正確に計算します。

### `CoinPhysics.cs`
各コインにアタッチされ、ライフサイクルと挙動を定義します。
- **設計意図**: 環境やスコアシステムに対するコインの反応を定義。
- **仕組み**:
    - **スコアリング**: `Finish`タグを持つオブジェクトとの接触を検知し、残高を増やして自身を破棄。
    - **親子関係**: プッシュ中の安定性を高めるため、`Pusher`への動的な親子付けを行います。

## ⚙️ セットアップ
正しく動作させるために、Unityプロジェクト内で以下のTagを定義する必要があります：
- `Pusher`: 動くプッシャーオブジェクトに設定。
- `Finish`: コインが回収されるトリガーエリアに設定。

### 操作方法
- **Spaceキー**: コインを投入。
